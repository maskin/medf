# MeDF - Meta/Modular e-Document Format

> **「思考とプロジェクトの"状態"を凍結・共有できる、AIネイティブなインデックス型ドキュメント」**

## 概要

MeDF（Meta/Modular e-Document Format）は、**AI時代に最適化された軽量・可変・機械可読なドキュメントフォーマット**です。

PDFの「紙前提・固定レイアウト・重量化」という制約から脱却し、以下を同時に実現します：

* ✅ **情報の全体像把握（インデックス性）** - 構造化されたメタデータとインデックスにより、AIとヒューマンの両方が迅速に全体像を理解
* ✅ **スナップショットとしての公式性** - 特定時点の状態を確実に記録・共有
* ✅ **分散管理された原本との整合性** - 外部ソースへの参照を保持し、常に最新状態を追跡可能

## MeDFが解決する課題

従来のドキュメントフォーマットには以下の問題がありました：

1. **全体像が誰にも見えていない** → 構造化メタデータとインデックスで解決
2. **正式情報と作業情報が混ざる** → レイヤー分離とステータス管理で解決
3. **AI時代の「記録単位」が存在しない** → MeDFが新しい標準単位に

## 主な特徴

### 🤖 AI最適化設計

* **構造化データ** - JSON形式により、AIが容易にパース・理解可能
* **多次元インデックス** - 目次、キーワード、エンティティ、関係性の4軸で索引化
* **セマンティック情報** - 明示的なメタデータにより文脈理解を支援

### 📊 柔軟な構造

* **モジュラー設計** - セクション単位での独立性と再利用性
* **バージョン管理** - セマンティックバージョニングによる変更追跡
* **外部参照** - 分散管理されたリソースへのリンクと整合性保持

### 🔒 信頼性とセキュリティ

* **不変性** - スナップショットは変更されない公式記録
* **アクセス制御** - メタデータレベルでの権限管理
* **完全性保証** - チェックサム・ハッシュによる改ざん検出

## ユースケース

* **プロジェクト状態レポート** - 特定時点での進捗状況、課題、決定事項の記録
* **AI対話の記録** - AIとの対話内容を構造化して保存・共有
* **分散ナレッジベースのスナップショット** - 複数ソースからの情報統合
* **仕様書・設計書の管理** - 技術ドキュメントのバージョン管理と承認プロセス

## クイックスタート

### 基本構造

MeDFドキュメントはJSON形式で記述されます：

```json
{
  "medf": {
    "version": "1.0",
    "schema": "https://medf.org/schema/v1.0"
  },
  "metadata": {
    "id": "unique-doc-id",
    "title": "ドキュメントタイトル",
    "type": "report",
    "status": "official",
    "created": "2026-02-03T23:00:00Z",
    "authors": [...],
    "language": "ja"
  },
  "index": {
    "toc": [...],
    "keywords": [...],
    "entities": [...],
    "relationships": [...]
  },
  "content": {
    "sections": [...]
  },
  "references": {
    "external": [...],
    "internal": [...]
  }
}
```

## ファイル命名規則

```
{document-name}.{version}.medf.json

例：
- project-status.v1.0.medf.json
- design-doc.v2.1.0.medf.json
- meeting-notes.v1.0.0.medf.json
```

## バージョニング

ドキュメントバージョンにはセマンティックバージョニング（Major.Minor.Patch）を使用します：

* **Major** - 構造の大幅変更
* **Minor** - セクションの追加・削除
* **Patch** - 内容の修正・更新
