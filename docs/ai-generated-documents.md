# AI生成文書（AI-generated documents）に対するMEDFの立場

## 1. 背景：なぜAI生成文書が問題になるのか

生成AIの普及により、以下のような課題が顕在化しています。

* **誰が・いつ・どのように作った文書か分からない**
* 内容が後から書き換えられても検出できない
* 人間が書いた文書とAI生成文書の区別がつかない
* 行政・論文・契約など「責任」が必要な文脈で使いづらい

重要なのは、問題の本質が **「AIで書いたこと」ではなく、
「来歴と責任が検証できないこと」** にある点です。

MEDF（Meaningful Document Format）は、
この構造的問題に対する **文書フォーマットとしての回答** を提供します。

---

## 2. MEDFの基本思想：AIを禁止しない

MEDFは、AI生成文書を **禁止・検出・排除** する立場を取りません。

代わりに、次の原則を採ります。

* AIで書いてもよい
* 人間が編集してもよい
* ただし **後から検証できる形で固定する**

MEDFにおいて重要なのは、
**「生成手段」ではなく「固定後の検証可能性」** です。

---

## 3. 文書の「固定」とは何か

MEDFでは、文書を以下のように扱います。

1. 文書を意味単位（blocks）に分解
2. 各ブロックと文書全体にハッシュ値を付与
3. そのハッシュに対して署名を行う

これにより：

* 内容が1文字でも変われば検出できる
* ブロック単位で引用・検証できる
* オフラインでも改ざん検出が可能

AI生成か人間生成かに関係なく、
**「その時点の文書内容」が証拠として残る** ことが保証されます。

---

## 4. AI生成であることはどこに記録されるのか

MEDFでは、AI利用の有無は **任意のメタデータ** として扱います。

例：

```json
"provenance": {
  "created_by": "human",
  "assisted_by": [
    {
      "type": "ai",
      "model": "gpt-4",
      "role": "drafting"
    }
  ]
}
```

重要なのは：

* この情報は**透明性のために記録できる**
* しかし **検証の必須条件ではない**
* 有無によって文書の有効性は変わらない

MEDFは「AI使用の告白フォーマット」ではありません。

---

## 5. なぜ電子透かしやAI検出に依存しないのか

AI検出・透かし技術には構造的な限界があります。

* モデル更新で検出精度が崩れる
* 翻訳・要約で消える
* オフラインでは成立しない
* ベンダー依存になる

MEDFはこれらを回避し、
**文書そのものの構造と暗号学的検証** によって信頼を成立させます。

---

## 6. 想定ユースケース

### 行政・公文書

* AIで下書き → 人間が確認 → MEDFで固定
* 後日「その時点の正式文書」を検証可能

### 論文・技術文書

* AI補助を使っても投稿可能
* 査読・引用は固定ブロック単位で可能

### AI時代のアーカイブ

* 「誰が書いたか分からないPDF」の氾濫を防ぐ
* 将来世代が来歴を検証できる記録を残す

---

## 7. MEDFが提供する答え

> **AI生成文書は危険なのではない。**
> **危険なのは、検証できない文書である。**

MEDFは、AI時代における文書の
**責任・来歴・検証可能性** を
フォーマットレベルで実装します。

---

## 8. まとめ（1行）

**MEDFは「AIを止める仕組み」ではなく、
「AI時代でも文書を信頼できるようにする仕組み」です。**
