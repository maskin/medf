# MeDF Examples

このディレクトリには、MeDFフォーマットの実際の使用例が含まれています。

## 含まれる例

### 1. project-status.v1.0.medf.json

プロジェクトの状態スナップショットの例です。

**特徴**:
- プロジェクトの進捗状況、課題、決定事項を記録
- 構造化データ（data型）と通常のテキストの混在
- 外部参照（JSON Schema、Semantic Versioning）へのリンク
- アノテーション（todo、note、info）の使用例

**学べること**:
- 複雑な階層構造の目次（TOC）の作成方法
- データ型のセクションの記述方法
- セクション間の関係性の定義
- 外部リソースへの参照方法

### 2. ai-dialogue.v1.0.medf.json

AI対話セッションの記録例です。

**特徴**:
- 質問と回答のペアを構造化して記録
- 対話から得られた知見の整理
- 複数の著者（Human + AI）の記録
- セクション内での構造化データの活用

**学べること**:
- 対話形式のコンテンツの記録方法
- データ型セクションでの複雑なオブジェクトの記述
- タイムスタンプの適切な使用
- 内部参照の活用

## 使い方

### 例を閲覧する

JSONファイルをテキストエディタやIDEで開いて閲覧できます。

```bash
# VS Codeで開く
code examples/project-status.v1.0.medf.json

# catで内容を表示
cat examples/ai-dialogue.v1.0.medf.json | jq .
```

### 検証する

バリデータースクリプトを使って、例が正しいMeDF形式であることを確認できます。

```bash
# すべての例を検証
python3 ../validate.py *.medf.json

# 特定の例を検証
python3 ../validate.py project-status.v1.0.medf.json
```

### テンプレートとして使う

これらの例をテンプレートとして、あなた自身のMeDFドキュメントを作成できます。

1. 例をコピーする
2. メタデータを更新（id, title, authors等）
3. 目次（TOC）を更新
4. コンテンツを書き換える
5. バリデータで検証する

## その他の例のアイデア

以下のような例も作成できます：

- **会議議事録**: 議題、討議内容、決定事項、アクションアイテム
- **技術調査レポート**: 背景、調査内容、比較表、推奨事項
- **API仕様書**: エンドポイント、リクエスト/レスポンス形式、認証
- **学習メモ**: 学習した内容、サンプルコード、参考資料
- **バグレポート**: 再現手順、環境情報、スタックトレース

あなたのユースケースに合った例を作成し、コミュニティと共有してください！
