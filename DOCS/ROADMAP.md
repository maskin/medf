# MeDF 開発ロードマップ

## 概要

本ロードマップは、MeDF (Meaning-anchored Document Format) の今後の開発計画を示します。

---

## フェーズ1: 基礎機能の強化 (v0.3 - v0.4)

### v0.3: 署名・検証機能の実装

**目標**: 公的文書としての信頼性を確保

- [ ] JWS (JSON Web Signature) 署名機能の実装
  - [ ] ES256/ES384/ES512 アルゴリズム対応
  - [ ] RS256/RS384/RS512 アルゴリズム対応
- [ ] 署名検証機能の実装
- [ ] DIDベースの署名者識別
- [ ] 署名タイムスタンプの対応
- [ ] 鍵管理機能の検討

**成果物**:
- CLIツールへの `sign` / `verify-signature` コマンド追加
- 署名付きサンプルファイル

### v0.4: バリデーションの強化

**目標**: MeDFファイルの品質保証

- [ ] セクション構造の検証
  - [ ] セクションIDの一意性チェック
  - [ ] セクション階層の整合性チェック
- [ ] Markdown拡張のパース
  - [ ] `medf-section` ブロックのパース
  - [ ] `reference` ブロックのパース
  - [ ] `note` ブロックのパース
- [ ] 必須フィールドの検証強化
- [ ] 参照URIの有効性チェック

**成果物**:
- 拡張バリデーション機能
- エラーメッセージの日本語化

---

## フェーズ2: 実用性の向上 (v0.5 - v0.6)

### v0.5: 変換機能の拡張

**目標**: 既存フォーマットからの移行を容易に

- [ ] PDF→MeDF 変換ツール
  - [ ] テキスト抽出
  - [ ] 構造認識（見出し、段落）
  - [ ] メタデータ抽出
- [ ] HTML→MeDF 変換ツール
- [ ] Word→MeDF 変換ツール
- [ ] 一括変換機能

**成果物**:
- `pdf2medf` コマンド
- `html2medf` コマンド

### v0.6: 差分・履歴管理

**目標**: ドキュメントの変更追跡

- [ ] MeDFファイル間の差分表示
- [ ] 変更履歴の自動生成
- [ ] セクション単位の変更追跡
- [ ] 視覚的な差分表示機能
- [ ] 変更のサマリー生成

**成果物**:
- `diff` コマンド
- `history` コマンド

## 長期的な展望

### 拡張機能の検討

1. **マルチリンガル対応**
   - 多言語コンテンツのサポート
   - 翻訳のリンク

2. **バイナリ添付**
   - 画像、図面の埋め込み
   - 添付ファイルの管理

3. **分散ストレージ連携**
   - IPFSとの統合
   - 永続的保存レイヤー

4. **AI連携**
   - 自動要約生成
   - 自動タグ付け
   - 関連ドキュメントの推薦

5. **Web UI**
   - ブラウザベースのビューア
   - 編集インターフェース
   - 署名・検証UI

---

## 優先順位の決定基準

1. **公的利用での必須要件**: 署名・検証
2. **移行の容易さ**: 変換ツール
3. **開発者体験**: パッケージ化・ドキュメント
4. **運用上の利便性**: 検索・差分
5. **将来の拡張性**: 高度な機能
